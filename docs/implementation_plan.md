# restapi-app の対話型処理への改修

いただいたご要望「`/dev` モデルリストを表示，ユーザが選択，選択したモデルをロード，ロードしたモデルでチャットと言う流れにして」に基づき、`main.py` を対話型のコンソールアプリケーションに変更する計画を作成しました。

## 実装要件
*   **モデルリストの取得と表示**: 起動時に `/api/v1/models` エンドポイントからモデル一覧を取得し、コンソールに番号付きで表示する。
*   **モデルの選択**: ユーザーに標準入力 (`input()`) で番号を指定させ、使用するモデルの `id` を決定する。
*   **モデルのロード**: 選択されたモデルをLM Studioサーバーでロードさせる。
*   **チャットの実行**: ロードされたモデルを使用し、指定したプロンプトでチャットを実行する（現状は固定のテスト文字列、あるいは対話的に入力を求めることも可能）。

## Proposed Changes

### [MODIFY] [main.py](file:///d:/WORK/AI/local_llm/restapi-app/main.py)
*   `fetch_models()` の結果を受け取り、コマンドライン上でモデル一覧 (`idx. model_id`) を表示。
*   `input("モデルの番号を選択してください: ")` 等を用いてユーザーのキー入力を待ち受ける処理を追加。
*   選択されたモデルについて、チャット実行の準備（または専用のLoad APIを用いたロード処理）を行う。
*   `chat_completion` 関数の呼び出し時、引数にユーザーのプロンプトと選択した `model_id` を渡す。
*   テストファイルの修正 (`tests/test_main.py` における標準入力 `input()` のモック対応など)。

## User Review Required & Open Questions

> [!IMPORTANT]
> **モデルの「ロード」に関するAPIの仕様確認**
> 現在のLM Studio REST APIにおいて、モデルを明示的にロードするための専用エンドポイント（例えば `POST /api/v1/models/load` など）をご存知でしょうか？
> 
> *   **A**: もし特に専用エンドポイントがない場合、LM Studioは**チャットリクエスト（`POST /api/v1/chat` に `model: "選択したモデル"` を含めたもの）を送るだけで自動的にロード**してくれる機能（JITロード）を持っています。この場合、チャット送信処理自体をロード処理として兼ねる実装とします。
> *   **B**: もしSDKのように「事前ロード用API」を使用するべきである場合は、そのエンドポイントのパス（例: `POST /api/v1/models` に `{ "model": "..." }` を投げる等）をご教示いただけますと幸いです。

上記について、現在のLM Studioサーバーの振る舞いと合わせてご確認いただき、A・Bどちらの方式で実装すべきかご指示（および承認）をお願いいたします。
